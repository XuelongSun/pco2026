### Lab 5 — CPU Hardware Architecture 1
### 实验5 — CPU硬件架构1

#### 1. 实验目标
完成CPU架构的第一部分的搭建，寄存器控制。

完成本实验后，能够：
1. 清楚数据总线的构建和时序控制逻辑
2. 构建系统时钟发生器：能够手动/自动产生时钟脉冲，能够产生复位信号
3. 构建多寄存器控制电路，通过指定寄存器的编号产生正确的寄存器控制信号
4. 构建带程序状态字（PSW）暂存功能的ALU

#### 2. 知识点
+ 总线：计算机系统中所有寄存器的数据输入和**带三态输出**的数据输出可以连接到同一个数据总线上
+ 寄存器控制器：输入读/写寄存器的编码，输出对应寄存器的控制信号
 
#### 3. 实验任务
##### 任务1：搭建系统时钟和电源控制器：
为了方便后续CPU调试与运行，先搭建一个系统脉冲和电源控制电路：

![](/Lab/Lab5/pow-pul-control.png)

+ 主体部分：2个反转(toggle)按钮：`POW`(power)和`MAN`(manual,手动)；2个一般按钮：`RES`(reset,复位)和`PUL`(pulse,手动产生脉冲)
+ 外部输入信号：`HLT`，`HLT`为1时，系统不再产生时钟信号。
+ 输出：复位信号，时钟脉冲

功能要求如下：
+ 上电后，复位信号输出保持为1
+ 当按下`POW`键后，复位信号输出为0，时钟脉冲持续输出
+ `POW`键保持按下，`MAN`按键也按下后，复位信号输出为0，时钟脉冲没有输出，此时，每点击一次`PUL`按钮便产生一个时钟脉冲（后续进行单步调试）。
+ 当输入`HLT=1`时，此时无论是手动模型(`MAN`按下)点按`PUL`还是自动模式（`MAN`没按下），都不会有时钟脉冲输出。

实现该电路并做验证。

##### 任务2：搭建多寄存器的数据通路
1. 利用任务1中所搭建的控制器，将所有寄存器的复位端口都连接到其复位信号输出上，时钟端口连接到其时钟脉冲输出上。然后将所有寄存器的数据输入和输出连接到一个8bit数据总线上（注意，寄存器必须有三态输出）。然后将各个寄存器的控制信号`WE`和`CS`拼成一个2bit的数值。
2. 封装一个8位地址位宽，8位数据位宽的RAM，构成一个`MC8X8`，将其也接入到我们的电路中。

最终形成类似如下的电路：

![](/Lab/Lab5/reg-group.png)

> 寄存器的数目需要大于等于10个

然后再来实现寄存器控制器：
+ 输入`W`：需要写入的寄存器编号（索引）
+ 输入`R`: 需要读出的寄存器编号（索引）
+ 输出所有寄存器的控制信号

![](/Lab/Lab5/reg-group-ctl.png)

功能要求：根据输入索引，产生正确的寄存器控制信号。例如`R=0x2`,`W=0x3`，则R2寄存器处在读状态（`CS=1`和`WE=0`），R3寄存器处于写状态（`CS=1`和`WE=1`），其余寄存器未选中（它们的`CS`和`WE`均等于0）

搭建上述电路，请根据寄存器的数量，自行决定输入`W`和`R`的位宽。并验证控制信号产生的正确与否。

##### 任务3：带PSW暂存的ALU
由于目前ALU电路是组合逻辑电路，无法保持之前的计算状态，为了让ALU也能参与到时序电路的控制，我们在ALU中加入一个寄存器，来暂存ALU的程序状态字（再次思考寄存器为什么叫寄存器）。

![](/Lab/Lab5/ALU-REG.png)

可采用上图电路加以验证: 上电之后寄存器值不确定，因此必须首先按一下Reset按钮输入到ALU新增的RT输入以给其内部寄存器清零。此ALU的`S`输出（即PSW）只有在输入`PE=1`时，且在时钟输入`C>`上升沿时更新为此时ALU的PSW的值进入其内部寄存器。

你可以使用一个溢出计算的例子，比如 $8-10$ 来验证此功能：

![](/Lab/Lab5/ALU-REG-1.png)

要求在[Lab3](/Lab/Lab3/readme.md)所搭建的ALU的基础上，搭建此ALU电路，并测试其功能的正确性。
